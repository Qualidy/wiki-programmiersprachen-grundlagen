    HTML Grundlagen 4 – CSS Fundamentals¶

    Lernziele

    Nach diesem Kapitel kannst du …

    • CSS-Selektoren für gezieltes Styling verwenden.
    • Eigenschaften und Werte korrekt anwenden.
    • Box-Model verstehen und kontrollieren.
    • Farben und Schriftarten gestalten.
    • Klassen und IDs sinnvoll einsetzen.

    Einführung¶

    CSS (Cascading Style Sheets) ist die Sprache für das Design von Webseiten: - Struktur → HTML - Design → CSS - Interaktion → JavaScript

    Wichtige Fakten: - CSS wurde 1996 veröffentlicht - Cascading: Regeln werden vererbt - Selektoren: Bestimmen, was gestylt wird - Box-Model: Grundlage für Layout

    Das Box-Model¶

    Grundprinzip¶

    Jedes HTML-Element ist eine Box: - Content: Der Inhalt - Padding: Innenabstand - Border: Rahmen - Margin: Außenabstand

    1
2
3
4
5
6
    .box {
  width: 200px;              /* Content-Breite */
  padding: 20px;             /* Innenabstand */
  border: 5px solid black;   /* Rahmen */
  margin: 30px;              /* Außenabstand */
}

    Gesamtbreite: 200 + 20 + 20 + 5 + 5 + 30 + 30 = 310px

    Box-Sizing (Wichtiger Trick!)¶

     1
 2
 3
 4
 5

.box {
  width: 200px;
  padding: 20px;
  /* Gesamtbreite: 240px */
}

/* Modern (empfohlen!) */
* {
  box-sizing: border-box;
}

.box {
  width: 200px;
  padding: 20px;
  /* Gesamtbreite: 200px (Padding ist inklusive) */
}

.text {

.text {

  color: red;

  color: green;
  color: orange;
  color: purple;
  color: gray;
  color: transparent;    /* Vollständig transparent */

    Häufige Fehler & Best Practices¶

    Fehler 1: Falsche Selektoren¶

    1
2
3
4
5
6
7
8
9
    /* ❌ Schlecht (zu spezifisch) */
div#header.container > ul li a.link {
  color: blue;
}

/* ✅ Gut (einfach und spezifisch) */
#header .nav-link {
  color: blue;
}

    Fehler 2: !important überall¶

    1
2
3
4
5
6
7
8
9
    /* ❌ Schlecht */
.text {
  color: red !important;
}

/* ✅ Gut (bessere Selektoren) */
.text.important {
  color: red;
}

    Fehler 3: Kein Box-Sizing¶

    1
2
3
4
5
6
7
    /* ❌ Falsch (veraltete Box-Model) */
/* Keine box-sizing Regel */

/* ✅ Gut (moderne Box-Model) */
* {
  box-sizing: border-box;
}

    Best Practices¶

    1. Externe CSS-Dateien verwenden
    2. Klassen statt IDs für Styling
    3. Einfache Selektoren (nicht zu spezifisch)
    4. Box-Sizing: border-box global setzen
    5. Mobile-First Ansatz
    6. Konsistente Namenskonventionen (BEM, camelCase)

    Zusammenfassung & Checkliste¶

    Wichtige Konzepte¶

    • ✅ Selektoren: Element, Klasse, ID, Attribute, Pseudo-Klassen
    • ✅ Box-Model: Content, Padding, Border, Margin
    • ✅ Farben: Hex, RGB, RGBA, HSL, Named
    • ✅ Schriftarten: Family, Size, Weight, Style
    • ✅ Hintergründe: Farben, Bilder, Gradienten
    • ✅ Display: Block, Inline, Inline-Block, Flex, Grid
    • ✅ Position: Static, Relative, Absolute, Fixed, Sticky

    Checkliste für CSS¶

    • [ ] Externe CSS-Datei verwendet
    • [ ] Box-Sizing: border-box gesetzt
    • [ ] Klassen statt IDs für Styling
    • [ ] Mobile-First Ansatz
    • [ ] Konsistente Namenskonventionen
    • [ ] Kommentare für komplexe Regeln
