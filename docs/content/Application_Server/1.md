## Einführung

Ein Anwendung‑Server ist die Laufzeitumgebung, in der Web‑Applikationen ausgeführt werden. Er verarbeitet HTTP‑Anfragen, bindet auf Ports, führt Code aus und liefert Antworten an Clients.

## HTTP Grundlagen

* **Request**: Methode (GET, POST, PUT, DELETE), Header, Body.
* **Response**: Status‑Code (200, 404, 500), Header, Body.
* **HTTPS**: TLS‑Handshake, Zertifikate, Cipher‑Suite.

## Deployment mit Docker

```dockerfile
# Base image
FROM openjdk:17-jdk-slim

# Arbeitsverzeichnis
WORKDIR /app

# Kopieren der JAR
COPY target/myapp.jar app.jar

# Exponieren des Ports
EXPOSE 8080

# Startbefehl
ENTRYPOINT ["java", "-jar", "app.jar"]
```

Build & Run:

```bash
docker build -t myapp .
docker run -d -p 8080:8080 myapp
```

## CI/CD‑Pipeline

1. **Build**: Kompilieren und Paketieren.
2. **Test**: Unit‑Tests, Integration‑Tests.
3. **Static Analysis**: SonarQube, SpotBugs.
4. **Deploy**: Push in ein Artifact‑Repository, Docker‑Image in ein Registry.
5. **Release**: Git‑Tag, automatisches Deployment in Staging/Production.

---

## Sicherheit & Skalierung

- **HTTPS‑Einrichtung**: Zertifikate von Let’s Encrypt.
- **CORS**: Nur vertrauenswürdige Origins erlauben.
- **Rate‑Limiting**: Mit Nginx/Traefik.
- **Horizontal Scaling**: Kubernetes Pods, Docker‑Swarm.
- **Logging**: Zentralisiert mit ELK oder Loki.
